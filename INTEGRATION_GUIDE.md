## Key Integration Points

### 1. JWT Authentication
- **Shared Secret**: Both services use the same `JWT_SECRET` environment variable
- **Token Format**: Normalized user fields (`user_id`, `userId`, `userID`) for compatibility
- **Generation**: Use `generateToken()` from auth modules
- **Verification**: Both services can verify tokens generated by each other

### 2. Database Configuration
- **Primary Database**: PostgreSQL (shared backend)
- **Tables**: Conversations, Messages, User Sessions, Activity Logs
- **Schema Initialization**: Automatic via database-migration.js in cloud-backend
- **Connection**: Pooled connections with configurable min/max

### 3. CORS Configuration
- **Development**: `http://localhost:3000,http://localhost:8000,http://localhost:8080,*`
- **Production**: Configure via `CORS_ORIGIN` environment variable
- **Examples**:
  - Local: `http://localhost:3000,http://localhost:8080`
  - Remote: `https://app.example.com,https://api.example.com`

### 4. WebSocket Integration
- **Authentication**: JWT token via headers or URL params
- **Heartbeat**: 30-second ping/pong mechanism
- **Session Management**: Redis-backed user-to-container mapping

## Local Development Setup

### Prerequisites
- Node.js >= 18.0.0
- PostgreSQL 16+
- Redis 7+
- Docker & Docker Compose (for containerized setup)

### Option 1: Native Installation

1. **Setup BE-Project**
   ```bash
   cd BE-Project
   cp .env.example .env
   npm install
   npm run dev
   ```

2. **Setup Cloud Backend**
   ```bash
   cd cloud-backend
   cp .env.example .env
   npm install
   npm run migrate  # Initialize database schema
   npm run dev
   ```

3. **Verify Integration**
   ```bash
   # Terminal 1 - BE-Project
   curl http://localhost:8080

   # Terminal 2 - Cloud Backend
   curl http://localhost:3000/api/health
   ```

### Option 2: Docker Compose (Recommended)

1. **Start All Services**
   ```bash
   cd cloud-backend
   docker-compose up -d
   ```

2. **Verify Services**
   ```bash
   # Check running containers
   docker-compose ps

   # Check logs
   docker-compose logs -f cloud-backend
   docker-compose logs -f be-project
   docker-compose logs -f postgres
   ```

3. **Access Services**
   - Cloud Backend API: `http://localhost:3000/api`
   - BE-Project API: `http://localhost:8080`
   - PostgreSQL: `localhost:5432` (postgres/postgres)
   - Redis: `localhost:6379`

No manual migrations needed for development.

## Authentication Flow

### Token Generation
```javascript
// Generate token (same across both services)
const token = generateToken({
  userId: 'user123',
  userName: 'john_doe',
  email: 'john@example.com'
});
```

### Token Verification
```javascript
// Verify in BE-Project
const decoded = jwt.verify(token, process.env.JWT_SECRET);

// Verify in Cloud Backend
const decoded = verifyToken(token);
```

### WebSocket Connection
```javascript
// Connect with token in header
const ws = new WebSocket('ws://localhost:3000', {
  headers: {
    'Authorization': `Bearer ${token}`
  }
});

// Or via URL parameter
const ws = new WebSocket(`ws://localhost:3000?token=${token}`);
```

## Troubleshooting

### JWT Token Errors
```
Error: Invalid token or token expired
```
**Solution**: Ensure `JWT_SECRET` is identical in both services

### CORS Errors
```
Access to XMLHttpRequest blocked by CORS policy
```
**Solution**: Add your domain to `CORS_ORIGIN` environment variable

### Database Connection Errors
```
Error connecting to database
```
**Solution**: 
- Verify PostgreSQL is running
- Check `DB_HOST`, `DB_USER`, `DB_PASSWORD`
- Ensure database exists

### WebSocket Connection Fails
```
WebSocket handshake: Unexpected response code 401
```
**Solution**:
- Verify JWT token is valid
- Check token is being sent in Authorization header
- Verify `JWT_SECRET` matches

## Security Best Practices

1. **Change Default Secrets**
   ```bash
   # Generate strong JWT secret
   openssl rand -base64 32
   ```

2. **Use HTTPS in Production**
   - Configure reverse proxy with SSL
   - Update WebSocket connections to `wss://`

3. **Database Security**
   - Use strong passwords
   - Restrict database access
   - Enable query logging

4. **API Security**
   - Rate limiting on endpoints
   - Input validation on all endpoints
   - CSRF protection enabled

5. **Monitoring**
   - Monitor authentication failures
   - Track resource usage
   - Alert on errors
